# Weekly Digest Template

Render the digest with this structure. Keep tables compact; drop sections
that have no data (and say so in one line instead).

```markdown
# 每周晨报 · 邮件 + 日程（或 Weekly Briefing · Mail + Calendar）

- 生成时间：<run timestamp with timezone>
- 邮件统计窗口：<last Mon 00:00> → <this Mon 00:00>
- 日程统计窗口：<today 00:00> → <+7d 00:00>
- 全程只读，未做任何写操作

## ① 上周邮件概况

| 指标 | 数值 |
|---|---|
| 邮件总量 | N 封 |
| 未读 | N 封（PCT%） |

| 发件人 | 数量 | 类型 |
|---|---|---|
| sender-1 | n | 分类（通知/告警/订单/人） |

## ② 疑似待回复（≤5）

1. **发件人** — 主题（日期）：为什么值得回（一句话）。

## ③ 本周日程

### 周一 MM-DD
- HH:mm 事件标题（地点/链接如有）

（无日程的星期写「无安排」）

## ④ 本周关注点（1–3 条）

- 把邮件线索与即将到来的日程交叉关联得出的建议。
```
